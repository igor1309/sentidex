**Что хранит в себе Claude Code Templates?
**
Я уже пару раз упоминал в постах репозиторий [CC Templates](https://github.com/davila7/claude-code-templates) от **AITMPL**. 

Репозиторий – это **сборник кастомных артефактов для Claude Code**. Агенты, команды, настройки, хуки, MCP-сервера (не собственные, но **хороший проверенный сборник**), плагины, скиллы. 

Также у них есть [удобный сайт](https://www.aitmpl.com/), через который можно изучить всё перечисленное.

Если у CC выходит какая-то новая абстракция – **в репозитории и на сайте примеры её использования появляются в тот же день**.

**Что с этим делать?**

Основное, для чего стоит использовать шаблоны – **быстрый прототип системы AI-агентов**. Которая, безусловно, **должна быть доработана под индивидуальные нужды пользователя**. 

Отдельно стоит отметить **собственные Tools для Claude Code, которые также есть на сайте** AITMPL. 

**Инструментов всего четыре:**

1. Claude Code Analytics.

2. Claude Code Health Check.

3. Claude Conversation Monitor.

4. Plugin Dashboard.

Мне **больше всего зашли Analytics и Conversation Monitor**. Первый позволяет через приятный интерфейс изучить, **куда и как были потрачены токены**. Чтобы, например, **оптимизировать сабагентов, отъедающих большую часть вашего лимита**.

**Conversation Monitor – это вообще must have tool**. Через него можно проштудировать уже состоявшиеся диалоги с Claude Code, чтобы понять, **где ваши AI-агенты шли по ложному пути или вызывали инструмент неправильно**. Отличный tool, чтобы **доработать чужих AI-агентов под собственные нужды**.

Health Check – **проверяет готовность всех компонентов текущего репозитория к разработке**. Хорош, когда надо разово проверить, что ничего из настроек не забыли (заметили, да, как **CC превратился в пульт управления космическим кораблём**?).

Plugin Dashboard – просто ещё один **удобный интерфейс**, чтобы устанавливать в свой Claude Code **плагины от CC Templates**.
