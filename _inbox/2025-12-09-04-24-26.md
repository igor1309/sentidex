**Думаю, стоит немного раскрыть концепции по работе с контекстом со схемы ****из вчерашнего поста**а](https://t.me/kdoronin_blog/1093)**.**

Начну с Scratchpad. 

Scratchpad – это часть контекста AI-агента, в которой **в структурированном виде хранится самая важная информация** о сессии. 

Структура для блокнота очень важна. И для её сохранения следует использовать Structured Output. Определяется структура архитектурой вашего AI-агента. 

Что такое **"самая важная информация" для AI-агента**?

**Например:**

1. Выжимка запроса пользователя.

2. Саммари по данным, которые вернули tools.

3. To-do-list с предстоящими этапами выполнения задачи.

4. Выводы, которые сделал AI-агент, принятые решения, а также шаги, которые уже совершил. **Кратко**.

Вот пример того, как запрос к AI-агенту на отдых можно структурировать и записать в Scratchpad:

```json

{
  "goal": "Спланировать 5-дневную поездку в Токио в мае",
  "user_prefs": {
    "budget": "средний",
    "interests": ["еда", "технологии"],
    "hate": ["толпы туристов"]
  },
  "itinerary": [],
  "facts": {
    "season": "май, высокий сезон, цветёт зелень",
    "weather": "18–23°C, возможны дожди"
  },
  "todos": [
    "Выбрать район проживания",
    "Разбросать по дням ключевые активности"
  ]
}

```
