✴️ **Обновление** **Codex 0.102: multi-agent и где это всё настраивается**

В **Codex 0.102** появилась **экспериментальная multi-agent поддержка**: теперь можно запускать несколько ролей параллельно прямо из терминального интерфейса. Включается в TUI через **/experimental**** → multi agents**, а сами настройки живут в конфиге Codex.

Где прописывать конфигурации:
 🟡 **Глобально для пользователя:** `~/.codex/config.toml` (то же самое, что `$CODEX_HOME/config.toml`, если вы меняли CODEX_HOME)
 🟡 **На уровне проекта (чтобы шарить в репозитории):** `.codex/config.toml `в корне репы или нужной папки
 🟡 **Отдельные файлы под роли агентов:** где угодно, но обычно удобно держать рядом, например `~/.codex/agents/<role>.toml` и ссылаться на них из `config.toml`  

Базовое включение фичи (в `~/.codex/config.toml` или `.codex/config.toml`):

```
[features]
multi_agent = true
```

Из коробки есть **3 роли**:
 🟡 **default** — смешанные задачи и дебаг: “`spawn default agent to debug the failure and propose fix`”
 🟡 **explorer** — ресерч по кодовой базе без правок: “`spawn explorer to map payment flow and check risks, no edits`”
 🟡 **worker** — кодинг и фиксы с тестами: “`spawn worker for src/auth/* and implement token refresh & run tests`”  

Кастомный агент (роль объявляете в `config.toml,` а настройки в отдельном файле):

```
[features]
multi_agent = true

[agents.fast_worker]
description = "Fast scoped implementation agent."
config_file = "/Users/<you>/.codex/agents/fast_worker.toml"
```

`~/.codex/agents/fast_worker.toml`:

```
model = "gpt-5.3-spark"
model_reasoning_effort = "low"
model_verbosity = "low"
```

Бонус: сейчас дефолт это **6 agent threads** на сессию. Меняется там же, в `config.toml`:

```
[agents]
max_threads = 12
```
