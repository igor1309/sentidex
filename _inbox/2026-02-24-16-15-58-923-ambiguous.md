Вау, а вот это классная штука, и странно, что мало кто пишет.

Anthropic выкатили **Programmatic Tool Calling** в GA. Раньше каждый вызов инструмента – отдельный запрос-ответ через модель. Claude вызвал API, получил результат, подумал, вызвал следующий. 20 вызовов = 20 проходов через модель. Дорого, медленно, контекст забивается сырыми данными.
Теперь Claude пишет Python-код(!), который вызывает ваши инструменты программно внутри sandbox-контейнера. Один проход модели → скрипт с циклами, фильтрацией, агрегацией → только финальный результат возвращается в контекст.

Как это выглядит:
— Добавляете allowed_callers: ["code_execution_20260120"] к определению инструмента
— Claude сам пишет код, который вызывает тулы как Python-функции
— Промежуточные результаты не попадают в контекст модели – только итог
— Контейнер живёт ~4.5 минуты и переиспользуется между запросами

Пример: нужно проверить выход за рамки бюджета 20 сотрудников. Классический подход – 20 запросов, тысячи строк в контексте. С PTC – один скрипт, цикл, фильтрация, на выходе только те, кто превысил лимит. У Anthropic в тестах расход токенов упал со 150K до 2K – в 75 раз(!). На практике экономия от 10x до 75x в зависимости от задачи.
На бенчмарках BrowseComp и DeepSearchQA именно добавление PTC поверх обычных тулов разблокировало полный потенциал агентов.
Где пригодится: batch-обработка в цикле (50 эндпоинтов за один вызов), early termination (нашёл ответ – остановился), условная логика (маленький файл → целиком, большой → summary), фильтрация (из 10K логов вернуть 10 последних ошибок).
Ограничения: не совместим со structured outputs (strict: true), нельзя форсировать через tool_choice, нужен code execution tool включённый.

Фича вышла в GA 17 февраля вместе с Sonnet 4.6. Работает только в API. Доступна для Claude Opus 4.6, Sonnet 4.6, Sonnet 4.5, Opus 4.5.

Если строите агентов на Claude API с множественными вызовами инструментов – это must-have. Особенно для сценариев с большими данными, где 90% информации модели вообще не нужно видеть.

Документация: [https://platform.claude.com/docs/en/agents-and-tools/tool-use/programmatic-tool-calling](https://platform.claude.com/docs/en/agents-and-tools/tool-use/programmatic-tool-calling)
