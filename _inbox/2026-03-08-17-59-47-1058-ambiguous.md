✴️ **/loop**** — новая команда в Claude Code для автоматических задач**

В **Claude Code** появилась новая функция **/loop**, которая позволяет запускать **повторяющиеся задачи прямо внутри сессии агента**. По сути это встроенный планировщик, который может **регулярно повторять любой промпт или команду** без ручного вмешательства.  

Функция подходит для ситуаций, где агент должен **следить за процессами, проверять обновления или автоматически реагировать на события** — например, контролировать PR, сборки или сообщения.

Как это работает:
 🟡 команда **/loop** создаёт фоновую задачу
 🟡 Claude автоматически **повторяет заданный промпт через заданный интервал**
 🟡 задачи выполняются **пока активна текущая сессия Claude Code**  

Примеры использования:
 🟡 /loop 5m check if the deployment finished — проверять статус деплоя каждые 5 минут
 🟡 /loop 20m /review-pr 1234 — каждые 20 минут запускать агент для ревью PR
 🟡 /loop babysit all my PRs — следить за PR и автоматически исправлять проблемы

Можно также использовать **внешние интеграции**, например:
 🟡 каждое утро собирать **дайджест упоминаний из Slack**
 🟡 автоматически **исправлять ошибки сборки**
 🟡 запускать **агентов для правки кода после комментариев в PR**

Как задаётся интервал:
 🟡 s — секунды
 🟡 m — минуты
 🟡 h — часы
 🟡 d — дни

Если интервал не указать, система **по умолчанию запускает задачу каждые 10 минут**.  

Есть несколько ограничений:
 🟡 задачи работают **только пока открыт Claude Code**
 🟡 они **не сохраняются после перезапуска**
 🟡 максимальное время жизни задачи — **до 3 дней**
 🟡 в одной сессии можно создать **до 50 задач**  

Под капотом используется обычный **cron-планировщик**, который запускает задачи в фоне и выполняет их между ответами агента.
