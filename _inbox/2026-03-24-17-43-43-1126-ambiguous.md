**5 граблей OpenClaw, на которые я наступил за вас**

Если ваш агент ведёт себя как на скриншотах — обещает сделать, переспрашивает, зависает и признаётся в тупости — скорее всего дело не в модели, а в конфигурации. Вот что помогло мне.

⚠️ Мои проблемы чаще всего происходят при исползовании прокси для ротации одноразовых ключей вместо опасного использования своей подписки

🧠 **Reasoning: модель «собирается сделать», но не делает**

Самая коварная проблема. Модель отвечает текстом, логи чистые — но инструменты не вызывает. `stopReason: stop` вместо `toolUse`.

Причина: нет флага `reasoning: true` в определении модели. Без него OpenClaw не шлёт reasoning-параметры в API.

```

❌ { "id": "gpt-5.3-codex", "name": "GPT-5.3 Codex" }
✅ { "id": "gpt-5.3-codex",  "name": "GPT-5.3 Codex" , "reasoning": true }

```

В `agents.defaults` ещё нужен уровень: `"thinkingDefault": "high" `— но без `reasoning: true` у вас `thinkLevel` принудительно переключится на off (см `src/agents/model-selection.ts`).

👁 **Работа с изображениями**

Отправляешь фото — I`mage model failed: 401`. Модель мультимодальая, но OpenClaw не знает, ищет отдельную модель для работы с изображениями.

Фикс: `"input": ["text", "image"]` в определении модели.

⏱ **Таймауты**

Дефолт 600 сек. Для `reasoning high` — мало. Ставьте `"timeoutSeconds": 900.`
И тут же попадайте на следующую проблему с блокировкой взаимодействия с ботом, пока он думает.

🔒 **Блокировки: бот думает и не отвечает**

OpenClaw обрабатывает сообщения последовательно. Пока агент думает 5–10 минут — входящие копятся. Агент не отвечает на сообщения. 

В `TOOLS.md` прописываем жёсткие правила делегирования. Всё, что больше двух вызовов инструментов: `sessions_spawn` в Codex workspace.

🔄 **Фидбек из субагентов: честный костыль**

`sessions_spawn` не блокирует диалог. Но результат не возвращается автоматически. Без напоминания агент забудет проверить.

`TOOLS.md`: после запуска задачи ставишь cron-напоминание через N минут. Нет результата, перенести на 5 минут. Тогда бот будет проверять результаты по расписанию.

Нативного колбека пока нет. Поэтому без такого костыля, костылем становится сам разработчик.

🎯 **Почему GPT-5.3 Codex, а не 5.4**

GPT-5.4 универсальная. Codex заточена под tool use. Если бот 80% времени дёргает инструменты — Codex справляется лучше. И Codex тоже мультимодальный. Короче у меня результьаты на кодексе как будто лучше, бот пытается угадывать действие, даже если оно не было явно указано в задаче

Ещё: `maxConcurrentSessions` в ACP — дефолт 4. У меня 40. Чтобы не упираться в лимит параллельных кодекс сессий.
