        1/ Мобильное приложение

        Знали, что у Claude Code есть мобильное приложение? Лично я пишу
        значительную часть кода прямо с iOS-приложения. Удобный способ вносить
        изменения без открытия ноутбука. Скачиваете Claude-приложение для
        iOS/Android, вкладка Code слева.


        2/ Переброс сессий между устройствами

        Перемещайте сессии между телефоном, вебом, десктопом и терминалом.
        Запустите claude --teleport или /teleport, чтобы продолжить облачную
        сессию на своей машине. Или /remote-control, чтобы управлять локально
        запущенной сессией с телефона/браузера. Лично я включил "Enable Remote
        Control for all sessions" в /config, чтобы каждая сессия по умолчанию
        была доступна удалённо.


        3/ /loop и /schedule

        Две самые мощные функции в Claude Code. Позволяют запланировать
        автоматический запуск Claude с заданным интервалом, до недели подряд. У
        меня локально крутится несколько циклов:


        /loop 5m /babysit - автоматически обрабатывает фидбек от code review,
        делает rebase и следит за CI

        /loop 30m /slack-feedback - каждые 30 минут собирает обратную связь из
        Slack и оформляет в PR


        4/ Hooks - детерминированная логика в жизненном цикле агента

        Используйте hooks, чтобы встраивать свою логику на определённых этапах
        работы агента. Примеры:


        Автоматически подгружать контекст при каждом старте Claude
        (SessionStart)

        Логировать каждую bash-команду, которую запускает модель (PreToolUse)

        Роутить запросы на подтверждение в WhatsApp, чтобы одобрять/отклонять с
        телефона (PermissionRequest)

        Автоматически «подталкивать» Claude продолжать работу, когда он
        останавливается (Stop)


        5/ Cowork Dispatch

        Использую Dispatch каждый день, чтобы разгребать Slack и почту,
        управлять файлами и делать дела на ноутбуке, когда я не за компьютером.
        Когда не кодирую - использую Dispatch. Это защищённый удалённый пульт
        для Claude Desktop. Может использовать ваши MCP-коннекторы, браузер и
        управлять компьютером.


        6/ Chrome-расширение для фронтенд-работы

        Главный совет по использованию Claude Code: дайте Claude способ
        верифицировать свой результат. Как только у него появляется обратная
        связь, качество результата вырастает в 2-3 раза. Представьте обычного
        инженера: если попросить его сделать сайт, но не дать открыть браузер -
        результат будет так себе. Дайте ему браузер - и он будет итерировать,
        пока не сделает хорошо. Chrome-расширение делает именно это для Claude.


        7/ Claude Desktop для автоматического запуска и тестирования
        веб-серверов

        В том же ключе - десктоп-приложение умеет автоматически запускать ваш
        веб-сервер и тестировать его во встроенном браузере. Аналогичное можно
        настроить в CLI или VS Code через Chrome-расширение.


        8/ Форк сессии

        Часто спрашивают, как разветвить текущую сессию. Два способа:

        Выполнить /branch внутри сессии

        Из CLI: claude --resume <session-id> --fork-session


        9/ /btw для боковых вопросов

        Использую постоянно, чтобы задавать быстрые вопросы, пока агент работает
        над основной задачей.


        10/ Git worktrees

        Claude Code поставляется с глубокой поддержкой git worktrees. Worktrees
        необходимы для массивной параллельной работы в одном репозитории. У меня
        одновременно запущены десятки Claude - и worktrees позволяют это делать.
        Используйте claude -w, чтобы запустить новую сессию в отдельном
        worktree.


        11/ /batch для масштабных изменений

        /batch проведёт интервью с вами, а затем раскидает работу на столько
        worktree-агентов, сколько нужно - десятки, сотни, даже тысячи.
        Используйте для больших миграций кода и других задач, которые можно
        распараллелить. Например: /batch migrate from jest to vite.


        12/ --bare для ускорения SDK до 10x

        По умолчанию при запуске claude -p (или через TypeScript/Python SDK)
        система ищет локальные [CLAUDE.md](CLAUDE.md), настройки и MCP. Но для
        неинтерактивного использования обычно нужно явно указать, что загружать.
        --bare пропускает автоматическую загрузку и ускоряет старт до 10 раз.
