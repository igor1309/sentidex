Я как-то писала про **Claude Code**e](https://t.me/NeuralProfit/1708),  сейчас вайбкодеры отдают предпочтение именно ему, говоря что **Claude Code** на уровень выше **Codex** или **Gemini**. Поэтому я собрала всю полезную информацию про **Claude Code**. 

🪼 **Обучалки:**

🤥 Помните, 🖥 **Anthropic** выложили курс для **Claude Code для начинающих**х](https://anthropic.skilljar.com/claude-code-in-action)? Там 15 лекций ~1 час видео + квиз еще и сертификат в конце выдадут. Этот курс стартует с базового уровня (*что такое ***Claude Code**e, как настроить, как работать с контекстом*), но дальше быстро *уходит в MCP, интеграцию с ***GitHub**b для автоматических ревью и обработки багов, hooks для добавления собственного поведения и проверок.* По сути курс с базового переходит к среднему уровню.

🤥 Далее, есть **Видеоурок про** **правильный старт проекта с Claude Code**e](https://www.youtube.com/watch?v=aQvpqlSiUIQ&feature=youtu.be), само обучение на **YouTube**, а значит можно посмотреть с переводом в **Яндекс Браузере** - за 30 минут автор показывает свою **схему PSB** **(***Plan → Setup → Build*ld)**: сначала нормально формулирует задачу и план работ, затем настраивает проект/окружение/правила, только потом пошагово делает фичи и правки.

🤥 **Глубокий гайд по Claude Code 2.0 + контекст-инжиниринг. **В виде **гайда**а](https://sankalp.bearblog.dev/my-experience-with-claude-code-20-and-how-to-get-better-at-using-coding-agents/#feature-deep-dive) есть мощный разбор от *sankalp*: что реально поменялось в **Claude Code 2.0** и как прокачать работу с агентами так, чтобы они  быстрее приносили результат. **Внутри:**
 •  Workflow “*plan → execute*” + как правильно стартовать проект ([CLAUDE.md](CLAUDE.md), scratchpad и т.п.)
 •  Команды/шорткаты и кастомные команды - чтобы повторяющиеся действия делались в 1 шаг
 •  Sub-agents (в т.ч *Explore/read-only*) - разделение задач и разгрузка контекста
 •  Context engineering - как экономить токены и ускорять работу

🤥 Если хотите понять “**почему агент ведет себя так”: system prompts Claude Code**. **Репозиторий**й](https://github.com/Piebald-AI/claude-code-system-prompts) с частями **системного промпта Claude Code** + token count каждого блока + история изменений по версиям. Полезно, если вы хотите понять, почему агент ведет себя именно так, или что поменяли после апдейта, понять какие правила и инструкции вшиты в **Claude Code**. **Бонус**: у этих же ребят есть **CLI tweakcc**c](https://github.com/Piebald-AI/tweakcc), чтобы кастомизировать части системного промпта , а не вручную ковырять по файлам.
